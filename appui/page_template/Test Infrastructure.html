Deployment Request Form

The deployment request dialog will allow the user to submit a deployment request.

  1. All site, database upgrades and deployments require a deployment request form to be filled out and submitted to the CM team.
  2. All scheduled deployments will be implemented on demand at a scheduled time.
  3. The CM team will notify the requestor of their deployment via MS Team channel.
  4. Emergency changes to arise and those requests still require a deployment request form. Prior to deployment, they must be reviewed by CM team.
  5. Required fields, Pull request ID’s, Target, Department, and Priority. Any error messages for the field will be immediately added or removed without clicking the submit button.
  6. All mandatory fields on forms are required. Form validation script in this case simply checks that a value has been entered in each of the form fields. Once all fields have been completed, the submit button would change color (washed out grey to blue) and become usable. After an entry is submitted, a success message will appear.
  7. The submitted request will create a team post in the “Deployment Request” channel.
